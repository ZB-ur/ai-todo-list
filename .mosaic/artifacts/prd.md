## Goal

打造一款「轻量但有结构感」的待办事项应用，以 AI 智能规划为核心差异点，面向追求效率但厌倦复杂工具的个人用户，提供移动优先、离线可用的任务管理体验。

## Features

- **F1: 任务 CRUD**：创建、编辑、删除、完成任务，支持任务标题、描述、截止日期、优先级（高/中/低）字段
- **F2: 智能列表与标签**：用户可创建自定义列表对任务分组；支持多标签标记；内置「今日」「即将到期」「已完成」智能列表自动聚合任务
- **F3: 自然语言输入**：支持自然语言快速创建任务（如「明天下午3点开会」自动解析为截止日期 + 标题），降低输入成本
- **F4: AI 任务拆解**：对复杂任务一键生成子任务建议，用户可编辑后确认，帮助用户将大目标分解为可执行步骤
- **F5: 离线优先 + 云同步**：本地优先存储，无网络时正常读写；联网后自动同步，冲突时以最后修改时间为准
- **F6: 用户认证**：支持邮箱注册/登录及 OAuth（Google）登录，JWT token 鉴权
- **F7: 响应式 UI**：移动优先设计，桌面端自适应布局；支持 PWA 安装到主屏幕
- **F8: 数据导入导出**：支持 CSV 格式导入/导出任务数据，降低迁移门槛

## Constraints

- **C1: 技术栈**：前端使用 React + Tailwind CSS；后端使用 RESTful API + PostgreSQL
- **C2: 移动优先**：所有 UI 设计以移动端为主视口（375px），桌面端为扩展适配
- **C3: 离线可用**：核心 CRUD 功能必须在无网络环境下正常工作，使用 IndexedDB 本地存储
- **C4: API 响应时间**：所有 API 端点 P95 延迟 ≤ 200ms
- **C5: AI 调用隔离**：AI 功能（F3 自然语言解析、F4 任务拆解）失败时不影响核心 CRUD 流程，降级为手动输入
- **C6: 单用户产品**：MVP 阶段不含协作/共享功能，仅支持个人使用

## Out of Scope

- **OS1: 团队协作**：多人共享列表、任务分配、评论等协作功能不在 MVP 范围
- **OS2: 日历视图/看板视图**：MVP 仅提供列表视图
- **OS3: 重复任务/习惯追踪**：周期性任务和习惯养成功能推迟到后续版本
- **OS4: 第三方集成**：与 Slack、Outlook、Google Calendar 等外部服务的集成不在 MVP 范围
- **OS5: 原生移动应用**：MVP 使用 PWA，不开发 iOS/Android 原生应用
- **OS6: 国际化**：MVP 仅支持中文界面
- **OS7: 付费功能分层**：MVP 阶段全部功能免费，不含订阅/付费机制
