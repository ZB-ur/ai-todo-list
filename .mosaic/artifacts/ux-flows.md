## User Journeys

### Flow 1: Basic Arithmetic Calculation
1. 用户打开计算器，显示屏表达式区为空，结果区显示 `0`
2. 用户点击数字按钮（或按物理键盘）输入第一个操作数 → 结果区实时更新当前数字
3. 用户点击运算符按钮（+、−、×、÷）→ 表达式区显示 `操作数 运算符`，结果区保持显示上一个数字
4. 用户输入第二个操作数 → 结果区实时更新当前输入数字
5. 用户点击 `=` 或按 Enter → 表达式区显示完整表达式（如 `12 + 8 =`），结果区显示计算结果 `20`
6. 用户可继续输入运算符进行连续运算（以上一次结果为起点），或输入新数字开始全新计算

### Flow 2: Continuous Calculation
1. 接 Flow 1 步骤 5，用户在得到结果后点击运算符 → 表达式区显示 `上次结果 运算符`，继续运算
2. 用户输入新操作数并按 `=` → 显示新结果
3. 可无限链式运算

### Flow 3: Decimal Input
1. 用户输入数字后点击 `.` → 结果区追加小数点
2. 继续输入数字 → 结果区显示完整小数
3. 若当前数字已包含小数点，再次点击 `.` 无效（静默忽略）

### Flow 4: Clear and Backspace
1. 用户点击 `AC` → 表达式区清空，结果区重置为 `0`，回到初始状态
2. 用户点击 `⌫` → 删除结果区最后一位字符；若删至空则显示 `0`
3. 在错误状态下点击 `AC` → 恢复正常状态

### Flow 5: Error Handling — Division by Zero
1. 用户输入 `5 ÷ 0 =`
2. 表达式区显示 `5 ÷ 0 =`，结果区显示错误提示 `不能除以零`
3. 用户按任意数字键或 `AC` → 清除错误，恢复正常状态

### Flow 6: Error Recovery
1. 计算器处于错误状态（如除以零后）
2. 用户按任意数字按钮 → 错误清除，开始新的输入
3. 用户按 `AC` → 错误清除，回到初始状态

### Flow 7: Keyboard Input
1. 用户按物理键盘 `0-9` → 等同于点击对应数字按钮
2. 按 `+` `-` `*` `/` → 等同于点击对应运算符按钮
3. 按 `Enter` → 等同于点击 `=`
4. 按 `Backspace` → 等同于点击 `⌫`
5. 按 `Escape` → 等同于点击 `AC`

## Interaction Rules

### 输入反馈
- 按钮点击时提供视觉反馈（按下态缩放或颜色变化），使用 Tailwind `active:` 伪类
- 物理键盘输入时对应按钮短暂高亮

### 显示规则
- 表达式区（上方）：较小字号，灰色文字，显示完整表达式历程
- 结果区（下方）：较大字号，主色文字，显示当前输入或计算结果
- 数字超长时自动缩小字号以适应显示区域（最多保留 10 位有效数字）
- 浮点修正：显示层对结果取 `toPrecision(10)` 后去除尾部多余零

### 运算符行为
- 连续点击运算符：替换为最后一次点击的运算符（不累加）
- 等号后直接输入数字：开始全新计算（清除表达式区）
- 等号后点击运算符：以当前结果为第一操作数继续运算

### 错误状态
- 错误信息以红色显示在结果区
- 错误状态下，除 `AC` 和数字键外，运算符按钮不可用（视觉置灰）
- 任意有效输入自动清除错误状态

### 按钮布局
- 4 列网格布局：
  - 第 1 行：`AC` `⌫` `÷` 
  - 第 2 行：`7` `8` `9` `×`
  - 第 3 行：`4` `5` `6` `−`
  - 第 4 行：`1` `2` `3` `+`
  - 第 5 行：`0`（跨 2 列） `.` `=`

### 响应式设计
- 桌面端：计算器居中显示，最大宽度 320px，带阴影卡片效果
- 移动端：计算器宽度自适应，按钮高度增大至 64px，确保触摸友好
- 所有按钮最小触摸目标 48×48px

## Component Inventory

- **Calculator**: 根容器组件，管理全局状态（表达式、当前输入、结果、错误状态），注册键盘事件监听
- **Display**: 显示区组件，包含表达式行和结果行，处理长数字自适应字号
- **ButtonGrid**: 按钮网格容器，4 列布局，管理按钮排列
- **CalcButton**: 单个按钮组件，接收 label / onClick / variant（number / operator / action / equals）/ span（列跨度），提供点击和键盘高亮反馈
- **ExpressionParser**: 工具模块（非 UI 组件），实现安全的四则运算解析，处理运算符优先级，不使用 eval
