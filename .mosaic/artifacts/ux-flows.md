## User Journeys

### Flow 1: Basic Calculation (basic-calc)
1. 用户看到计算器初始状态，显示屏显示 `0`
2. 用户点击数字键（如 `5`），显示屏更新为 `5`
3. 用户点击运算符（如 `+`），运算符高亮，显示屏保持 `5`
4. 用户输入第二个操作数（如 `3`），显示屏更新为 `3`
5. 用户点击 `=`，显示屏显示结果 `8`
6. 结果 `8` 作为下一次运算的第一个操作数，可继续输入运算符

### Flow 2: Multi-digit & Decimal Input (digit-decimal)
1. 用户连续点击 `1` `2` `3`，显示屏依次更新为 `1` → `12` → `123`
2. 用户点击 `.`，显示屏更新为 `123.`
3. 用户点击 `4` `5`，显示屏更新为 `123.45`
4. 用户再次点击 `.`，无响应（同一数字仅允许一个小数点）

### Flow 3: Chained Operations (chained-ops)
1. 用户输入 `3`，点击 `+`
2. 用户输入 `5`，点击 `×`
3. 此时先计算 `3 + 5 = 8`，显示屏显示 `8`，运算符切换为 `×`
4. 用户输入 `2`，点击 `=`
5. 计算 `8 × 2 = 16`，显示屏显示 `16`

### Flow 4: Clear / Reset (clear-reset)
1. 用户在任意状态下点击 `AC`
2. 显示屏归零为 `0`，所有运算状态清除
3. 计算器回到初始状态

### Flow 5: Divide by Zero (divide-by-zero)
1. 用户输入 `7`，点击 `÷`，输入 `0`，点击 `=`
2. 显示屏显示 `Error`
3. 此时仅 `AC` 键可用，其他按键无响应
4. 用户点击 `AC`，计算器恢复初始状态

### Flow 6: Overflow Display (overflow-display)
1. 用户输入或计算出超过 12 位的数字
2. 显示屏自动切换为科学计数法（如 `1.23456e+15`）或缩小字体以适配显示区域
3. 用户仍可正常继续运算

## Interaction Rules
- **输入即时反馈**: 每次按键后显示屏立即更新，无延迟
- **运算符高亮**: 当前激活的运算符按钮保持高亮状态，输入下一个数字后取消高亮
- **重复等号**: 连续按 `=` 不重复计算，保持当前结果
- **运算符切换**: 连续点击不同运算符（如先 `+` 再 `-`），以最后一次为准，不触发计算
- **结果续算**: 按 `=` 得到结果后，直接点击运算符以结果为第一操作数继续运算
- **结果覆盖**: 按 `=` 得到结果后，直接输入数字则清除结果开始新运算
- **Error 锁定**: 显示 `Error` 时，仅 `AC` 键响应，其他按键忽略
- **前导零抑制**: 输入 `0` 后再输入其他数字，自动替换（`07` → `7`），但 `0.` 保留

## Component Inventory
- **Display**: 顶部显示区域，右对齐，展示当前数字/结果/Error，支持字体自适应缩小
- **DigitButton**: 数字键 0-9，标准网格按钮样式，`0` 占两列宽度
- **DecimalButton**: 小数点 `.` 按钮，与数字键同行
- **OperatorButton**: 运算符按钮（`+` `-` `×` `÷`），右侧一列，支持高亮激活态
- **EqualsButton**: 等号 `=` 按钮，醒目强调色
- **ClearButton**: AC 按钮，左上角位置
