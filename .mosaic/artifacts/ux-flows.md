## User Journeys

### Flow 1: Basic Calculation (basic-calc)
1. 用户打开计算器页面 → 显示空表达式区域，结果显示"0"
2. 用户通过按钮或键盘输入数字 → 表达式区域实时更新，结果区域显示预览
3. 用户输入运算符（+、-、×、÷） → 运算符追加到表达式
4. 用户继续输入第二个数字 → 结果区域实时预览计算结果
5. 用户按"="或 Enter → 表达式求值，结果显示在结果区域，记录写入历史
6. 用户可继续输入新运算符（基于上次结果链式计算）或按 C 清空重新开始

### Flow 2: Input Correction (input-correction)
1. 用户输入过程中发现错误 → 按 ⌫ 或 Backspace 删除最后一个字符
2. 表达式和预览结果实时更新
3. 若需完全重来 → 按 C 或 Escape 清空所有输入，回到初始状态

### Flow 3: History Recall (history-recall)
1. 用户完成若干次计算后 → 历史记录面板显示最近计算（表达式 = 结果）
2. 用户点击某条历史记录 → 该条结果回填到当前输入（作为新表达式的起始数字）
3. 用户可基于回填值继续计算

### Flow 4: Error Recovery (error-recovery)
1. 用户输入除以零的表达式（如 5÷0）→ 按 = 后结果区显示"错误"
2. 用户输入不完整表达式（如 5+）→ 按 = 后结果区显示"格式错误"
3. 错误状态下，用户按 C/Escape 清空 → 恢复正常状态，可重新输入
4. 错误状态下，用户直接输入数字 → 清空错误，开始新表达式

### Flow 5: Decimal & Sign Toggle (decimal-sign)
1. 用户按"."输入小数 → 当前数字追加小数点（同一数字中只允许一个小数点）
2. 用户按"±" → 当前数字正负切换
3. 结果显示遵循精度规则（最多 10 位有效数字）

## Interaction Rules

- **实时预览**: 每次输入变化时，若当前表达式可求值，结果区域立即显示预览值（灰色/半透明），按 = 后变为正式结果（黑色/全透明）
- **表达式长度限制**: 表达式达到 50 字符时，忽略后续字符输入，按钮/键盘无响应（不显示错误提示）
- **连续运算符防护**: 连续输入运算符时，新运算符替换旧运算符（如 5+- 变为 5-）
- **前导零处理**: 输入 00 自动归为 0，输入 05 自动归为 5（小数点前除外）
- **按钮反馈**: 按下时有视觉反馈（scale 缩小 + 背景色变化），持续 100ms
- **键盘映射**: 数字键 0-9 → 对应数字，+/-/\*/\ → 对应运算符，Enter → =，Backspace → ⌫，Escape → C
- **历史记录滚动**: 历史超出可视区域时可滚动查看，最新记录在顶部
- **localStorage 持久化**: 历史记录在每次计算完成后写入 localStorage，页面加载时读取恢复
- **大数显示**: 结果超出安全整数范围时自动切换科学计数法显示
- **错误不入历史**: 产生错误的计算不写入历史记录

## Component Inventory

- **Calculator**: 根容器组件，管理整体布局（居中卡片/移动端全屏），组合所有子组件
- **Display**: 显示区域，包含两行——上方表达式行（较小字号，可横向滚动）和下方结果行（较大字号）
- **ButtonGrid**: 按钮网格布局（4 列），包含数字键、运算符键和功能键，响应式调整按钮尺寸
- **Button**: 单个按钮组件，接收 label/onClick/variant（number/operator/function），提供点击动画反馈
- **HistoryPanel**: 历史记录面板，显示在计算器主体右侧（桌面端）或下方（移动端），可滚动列表
- **HistoryItem**: 单条历史记录，显示"表达式 = 结果"，可点击回填
