## User Journeys

### Flow 1: Basic Calculation
1. 用户看到计算器界面，显示区域初始值为 **0**
2. 点击数字键（0-9），显示区域实时更新为输入的数字
3. 点击运算符（+、−、×、÷），当前数字被暂存，显示区域等待新输入
4. 输入第二个数字
5. 点击 **=**，显示计算结果

### Flow 2: Chained Operations
1. 完成一次基础计算后（或输入第一个数字 + 运算符 + 第二个数字）
2. 不点击 =，直接点击下一个运算符
3. 系统自动计算前一步结果并显示
4. 用户继续输入下一个数字
5. 可重复步骤 2-4 进行连续运算
6. 点击 **=** 显示最终结果

### Flow 3: Decimal Input
1. 输入整数部分（或直接以 0 开头）
2. 点击 **.**，显示区域追加小数点
3. 继续输入小数位数字
4. 再次点击 **.** 无效（每个数字仅允许一个小数点）

### Flow 4: Error Recovery
1. 用户输入一个数字，点击 **÷**，输入 **0**
2. 点击 **=**，显示区域显示 **"Error"**
3. 用户点击 **AC** 或任意数字键，恢复正常状态（AC 重置为 0，数字键开始新输入）

### Flow 5: Clear & Backspace
1. **AC**：任何时候点击，清除所有状态，显示区域重置为 **0**
2. **退格**：删除显示区域最后一位字符；若删至空则显示 **0**；在 Error 状态下等同于 AC

### Flow 6: Long Number Display
1. 用户持续输入数字，显示区域实时更新
2. 数字长度超过显示区域宽度时，自动缩小字号以适配
3. 超过 16 位时，切换为科学计数法显示

## Interaction Rules

### 输入反馈
- 每次按键立即更新显示区域，无延迟
- 按钮按下时有视觉反馈（背景色变化）

### 数字输入
- 初始状态或运算符后，输入数字替换显示的 0（不追加）
- 非初始状态下，数字追加到当前显示值末尾
- 前导零自动消除：输入 "007" 显示为 "7"（小数点情况除外："0.07" 正常显示）

### 运算符行为
- 连续点击运算符：替换为最后一个运算符，不触发计算
- 在 = 结果后点击运算符：以当前结果为第一操作数继续运算
- 在 = 结果后点击数字：开始全新计算

### 等号行为
- 有完整表达式（数字 + 运算符 + 数字）时计算并显示结果
- 无运算符时点击 = 无效果
- 结果显示后再次点击 = 无效果

### Error 状态
- 显示 "Error" 时，所有运算符和 = 和 . 无效
- 点击 AC 重置为初始状态
- 点击数字键直接开始新输入（隐式清除错误）

### 显示溢出
- 结果超过 16 位有效数字时使用科学计数法（如 1.234567890123e+20）
- 输入过程中限制最多 16 位数字字符

### 响应式布局
- 桌面端：计算器居中显示，最大宽度 320px
- 移动端：按钮最小触控区域 44×44px，间距 ≥ 8px
- 显示区域高度固定，文字垂直居中靠右对齐

## Component Inventory

- **Display**: 顶部显示区域，展示当前输入值或计算结果，支持自动缩小字号和右对齐
- **NumberButton**: 数字键（0-9），0 键占两格宽，其余单格，统一圆角样式
- **OperatorButton**: 运算符键（+、−、×、÷），视觉上与数字键区分（高亮色）
- **EqualsButton**: 等号键（=），与运算符同列，视觉突出（主色调）
- **DecimalButton**: 小数点键（.），位于底行 0 键右侧
- **ClearButton**: AC 按钮，位于顶行左侧，清除所有状态
- **BackspaceButton**: 退格按钮（⌫），位于顶行，删除最后一位
- **ButtonGrid**: 按钮网格布局容器，4 列排列，管理按钮间距和响应式尺寸
- **Calculator**: 根容器组件，组合 Display 和 ButtonGrid，居中布局和阴影边框
