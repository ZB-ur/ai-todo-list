## User Journeys

### Flow 1: Basic Calculation
1. 用户看到计算器界面，显示屏显示 "0"
2. 用户点击数字按钮（0-9）输入第一个操作数 → 显示屏实时更新
3. 用户点击小数点（.）可输入小数 → 显示屏追加小数部分
4. 用户点击运算符（+、−、×、÷）→ 显示屏保留第一个操作数，运算符高亮
5. 用户点击数字按钮输入第二个操作数 → 显示屏显示第二个操作数
6. 用户点击等号（=）→ 显示屏显示计算结果

### Flow 2: Continuous Calculation
1. 上一次计算结果显示在屏幕上
2. 用户直接点击运算符 → 结果作为第一个操作数，进入运算符选择状态
3. 用户输入第二个操作数 → 显示屏显示新输入
4. 用户点击等号 → 显示新结果
5. 可无限重复此链式操作

### Flow 3: Clear and Backspace
- **AC（全部清除）**：任何状态下点击 AC → 重置为初始状态，显示屏归 "0"
- **⌫（退格）**：输入数字时点击 ⌫ → 删除最后一位；若删到空则显示 "0"；在结果/运算符状态下点击 ⌫ 无效果

### Flow 4: Error Recovery
1. 用户输入 `N ÷ 0 =` → 显示屏显示 "Error"
2. Error 状态下，数字和运算符按钮无响应
3. 用户点击 AC → 恢复初始状态，可继续使用

### Flow 5: Overflow Display
1. 计算结果超出常规显示位数 → 自动切换为科学计数法显示
2. 显示屏字体随位数增多自动缩小，确保不溢出容器

## Interaction Rules

- **输入即显**：每次按键立即更新显示屏，无延迟
- **重复小数点忽略**：当前数字已含小数点时，再次点击小数点无效
- **运算符替换**：连续点击不同运算符，以最后一次为准（如 `5 + −` 变为 `5 −`）
- **等号后输入数字**：直接开始新的计算，替换当前显示
- **等号后点运算符**：进入连续运算流程（Flow 2）
- **前导零处理**：输入 "0" 后直接输入非零数字，替换 "0"；输入 "0." 保留
- **字体自适应**：显示内容超过 8 位时字体逐步缩小，最小不低于 16px
- **触摸区域**：所有按钮最小触摸区域 44×44px，移动端适配
- **按钮反馈**：点击时提供视觉反馈（按下态颜色变化）

## Component Inventory

- **Display**：顶部显示区域，展示当前输入值或计算结果，支持字体自适应缩放
- **DigitButton**：数字按钮（0-9 和小数点），标准尺寸，0 按钮占两列宽
- **OperatorButton**：运算符按钮（+、−、×、÷），右侧一列，带高亮选中态
- **EqualsButton**：等号按钮，右下角，强调色背景
- **ClearButton**：AC 全部清除按钮，左上角功能区
- **BackspaceButton**：⌫ 退格按钮，功能区
- **ButtonGrid**：4 列按钮网格布局容器，管理按钮排列和间距
- **Calculator**：顶层容器组件，包含 Display + ButtonGrid，管理计算状态机
